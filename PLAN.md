# 音樂分源程式 規劃文件

## 專案概述

桌面應用程式，讓使用者上傳音樂檔案，自動分離各音軌並下載結果。

---

## 技術選型

### 分源模型：Demucs (htdemucs_6s)
- Facebook Research 開發，開源
- 支援 6 個音軌：**人聲、鼓、貝斯、吉他、鋼琴、其他**
- 「吉他」涵蓋電吉他與木吉他
- 「其他」涵蓋合成器、弦樂、鍵盤等其他流行樂器
- 安裝：`pip install demucs`

> 補充：若日後對鋼琴品質不滿意，可考慮疊加 Spleeter 做二次分離。

### 前端框架：PySide6（Qt for Python）
- 成熟穩定，跨平台（Windows / macOS / Linux）
- 支援現代 UI 設計
- 授權：LGPL

### 專案語言：Python 3.10+

---

## 支援的分離音軌

| 音軌      | Demucs stem | 說明                        |
|-----------|-------------|-----------------------------|
| 人聲      | vocals      | 主唱、和聲                  |
| 鼓        | drums       | 鼓組、打擊樂                |
| 貝斯      | bass        | 電貝斯                      |
| 吉他      | guitar      | 電吉他 + 木吉他             |
| 鋼琴      | piano       | 鋼琴、電鋼琴                |
| 其他      | other       | 合成器、弦樂、管樂等        |

---

## 功能規格

### 核心功能
1. **匯入音檔**：支援 MP3、WAV、FLAC、M4A
2. **選擇音軌**：勾選要分離的音軌（可全選）
3. **分離執行**：顯示進度條，背景執行
4. **預覽播放**：分離完成後可在程式內播放各音軌
5. **匯出音軌**：選擇輸出格式（WAV / MP3）與資料夾

### 進階功能（第二階段）
- 靜音特定音軌後重新混音輸出（卡拉 OK 模式）
- 調整各音軌音量後混音
- 批次處理多首歌曲

**各元件說明：**

| 元件 | 行為 |
|------|------|
| 整體播放鈕 | 所有音軌同步播放（依各軌音量/M/S 狀態混音） |
| 整體進度條 | 可拖曳 seek，所有音軌同步跳轉 |
| 整體音量 | 整體輸出增益（疊加在各軌音量之上） |
| 整體下載 | 依目前所有音量/靜音狀態混音，輸出 MP3 320kbps |
| 音軌 ▶ 播放 | 獨立播放該音軌（不影響其他軌、不走 M/S 邏輯） |
| M（Mute） | 整體播放時靜音該軌，channel 顯示灰色 |
| S（Solo） | 整體播放時只聽該軌，其餘靜音 |
| 音軌音量滑桿 | 0～150%，即時套用增益（整體播放 & 獨立播放皆有效） |
| 音軌下載鈕 | 依該軌目前音量輸出 MP3 320kbps（不含其他音軌） |
| 波形圖 | 顯示音訊振幅，整體播放時 playhead 同步移動 |

**MP3 編碼規格（所有下載統一）：**
- 格式：MP3
- 位元率：320 kbps（CBR）
- 工具：ffmpeg（透過 `ffmpeg-python` 或 `pydub` 呼叫）

## 注意事項

- Demucs 首次執行會自動下載模型（約 300MB）
- 分離速度取決於硬體：有 NVIDIA GPU 會快很多（CUDA）
- CPU 模式下一首 4 分鐘歌曲約需 3～10 分鐘
